# Ios Settings Patterns
**Purpose:** Apply ios settings patterns as a deliberate UX decision — informed by user research, platform conventions, and design principles.
## Core Framework
Platform conventions exist because millions of users learned them. Violating platform conventions creates friction. The skill is knowing which conventions to follow, which to adapt, and which to deliberately break.
## Principle
Every design decision should be traceable to a user need or a business objective. If the rationale for a decision is "it's best practice" or "it looks good," the decision hasn't been justified — it's been assumed.
## When to use
When this specific UX challenge is the active constraint.
