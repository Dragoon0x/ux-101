# Inclusive Language
**Purpose:** Use language that includes all users. Gender: use 'they/them' as singular pronouns, avoid 'guys' for mixed groups. Ability: 'accessible' not 'handicapped-accessible,' 'authentication' not 'sanity check.' Culture: avoid idioms that don't translate ('hit the nail on the head'). Technical level: define jargon on first use, or better, don't use it. Age: don't assume digital literacy or lack thereof. Test copy with diverse reviewers.

## When to use
When writing or reviewing this type of interface text. When the current copy is unclear, too long, or not helping users accomplish their goals.
