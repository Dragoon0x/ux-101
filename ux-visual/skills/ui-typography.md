# UI Typography
**Purpose:** Design typography for interfaces that is readable, hierarchical, and consistent — the invisible foundation that determines whether users can actually consume the content.

Typography accounts for 80-90% of most interfaces. The vast majority of what users see on screen is text. Yet typography is treated as an afterthought in most product design — defaults accepted, sizes arbitrary, spacing inconsistent. Professional UI typography creates a system where every text element communicates its purpose through its visual properties.

## The Type Scale
A type scale is a set of predetermined sizes that maintain mathematical and visual harmony. The most common approach is a modular scale — each size is a fixed ratio larger than the previous (1.2x, 1.25x, 1.333x). A disciplined scale prevents the creep of arbitrary sizes that makes an interface feel chaotic.

## The Readability Standard
Body text in interfaces should be 16px minimum on desktop, 16px on mobile. Line length should be 50-75 characters for comfortable reading. Line height should be 1.4-1.6 for body text, tighter for headings. These aren't aesthetic preferences — they're research-backed standards for readability.

## The Weight System
Use a maximum of 3-4 font weights in an interface. Regular for body text. Medium or semibold for emphasis and labels. Bold for headings. More weights create visual noise without adding communicative value.

## When to use
Every interface design. When text is hard to read. When the interface feels cluttered despite having enough space. When building or refining a design system's typography tokens.
