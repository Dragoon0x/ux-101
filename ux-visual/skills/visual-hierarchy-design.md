# Visual Hierarchy Design
**Purpose:** Control what users see first, second, and not at all — because visual hierarchy is the primary mechanism for communicating importance in an interface.

Visual hierarchy is the invisible hand that guides the user's eye through the interface. It determines what gets noticed, what gets read, and what gets ignored. Without deliberate hierarchy, users scan randomly and miss critical information. With deliberate hierarchy, users perceive information in the order the design intends.

## The Hierarchy Tools
Size, color, contrast, weight, spacing, and position are the six tools of visual hierarchy. Size and contrast are the strongest signals — large, high-contrast elements dominate attention. Position matters because users scan in predictable patterns (F-pattern for text-heavy content, Z-pattern for image-heavy content). Spacing creates grouping — elements close together are perceived as related.

## The Three-Level Rule
Most interfaces need exactly three hierarchy levels: primary (the one thing the user should do or see first), secondary (the supporting information and alternative actions), and tertiary (the details available on demand). More than three levels creates visual noise. Fewer than three creates ambiguity about what's important.

## The Squint Test
Squint at the interface until the details blur. What's still visible? That's your hierarchy. If everything blurs into the same level of prominence, the hierarchy is broken. If one element clearly dominates, the hierarchy is working.

## When to use
Every screen, every component, every design decision. Visual hierarchy is not a phase of design — it's a property of every design that is either intentional or accidental.
