# Navigation Design
**Purpose:** Design navigation systems that get users where they want to go with minimum effort and maximum confidence — because navigation is the primary interface between user intent and product content.

Navigation is the most-used and least-designed element of most products. It's treated as a container for links rather than a communication system. Good navigation does three things simultaneously: it tells users where they are, it tells them where they can go, and it tells them how to get back. Missing any of these three creates disorientation.

## The Navigation Types
Global navigation persists across the product and provides orientation. Local navigation shows options within a section. Contextual navigation connects related content. Utility navigation provides account and system functions. Each type serves a different need and they work together as a system.

## The Orientation Principle
Users should always be able to answer three questions: where am I? how did I get here? where can I go? If the navigation doesn't answer all three, users will feel lost even if they're technically able to click through the interface.

## The Mobile Navigation Challenge
Mobile navigation has less space and higher stakes — users on mobile are more likely to be task-focused and less tolerant of exploration. The hamburger menu hides everything. Bottom navigation shows everything but limits items. The right pattern depends on how many primary destinations the product has and how often users switch between them.

## When to use
When designing navigation for a new product. When navigation is the source of user confusion. When the product grows and the original navigation can't accommodate new content. When redesigning for mobile or responsive contexts.
