# Faceted Navigation Design
**Purpose:** Design faceted filtering that lets users narrow large sets of content — the patterns and pitfalls of multi-attribute filtering.
## Core Framework
If users can't find it, it doesn't exist. Information architecture is the invisible structure that makes complex systems navigable.
## Principle
Every design decision should be traceable to a user need or a business objective. If the rationale for a decision is "it's best practice" or "it looks good," the decision hasn't been justified — it's been assumed.
## When to use
When this specific UX challenge is the active constraint.
