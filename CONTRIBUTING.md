# Contributing

**ux-101** is opinionated by design. The bar is the same as every repo in this ecosystem — high.

## The bar

- Every skill encodes real UX judgment from people who've designed real products used by real people.
- Specific over generic. "Test with users" is not a skill. "Five users find 85% of usability problems, but that's for qualitative pattern detection — measuring satisfaction requires statistical rigor and a larger sample" is.
- No UX platitudes. No frameworks named after shapes that exist to sell consulting.
- If the skill could appear in a generic UX tips listicle without standing out, it's not ready.

## How to contribute

1. Fork the repo
2. Create a branch: `git checkout -b skill/your-skill-name`
3. Follow the SKILL.md format in the relevant domain
4. Open a PR with one line: what UX judgment does this skill encode?
