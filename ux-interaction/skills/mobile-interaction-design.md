# Mobile Interaction Design
**Purpose:** Design interactions that work within the constraints and opportunities of mobile — small screens, touch input, interrupted use, and one-handed operation.

Mobile is not desktop shrunk down. It's a fundamentally different context with different input methods, different attention patterns, and different ergonomic constraints. Designing for mobile means designing for thumbs, not cursors. For glances, not sessions. For interruption and resumption, not continuous focus.

## The Thumb Zone
On a phone held in one hand, the thumb can comfortably reach about 60% of the screen. The bottom center is the easiest zone. The top corners are the hardest. Primary actions should live in the easy zone. Navigation that requires reaching to the top of a tall phone creates ergonomic strain that users feel as friction.

## The Touch Target
The minimum touch target is 44x44 points (Apple) or 48x48dp (Google). This is a minimum, not a target — larger is always better for frequently used actions. Targets that are too close together cause mis-taps, which cause errors, which cause frustration. Spacing between targets matters as much as size.

## The Gesture Vocabulary
Users have learned a standard gesture vocabulary: tap, swipe, pinch, long-press, pull-to-refresh. Using these gestures for their expected purposes works well. Inventing new gestures or repurposing standard gestures creates a learning burden that most users won't invest in.

## When to use
Every mobile design. When adapting a desktop product for mobile. When mobile usage metrics indicate interaction problems. When designing touch-first interfaces.
