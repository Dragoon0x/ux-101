# Modal & Dialog Design
**Purpose:** Design modals that serve users instead of trapping them — when to use dialogs and when to use in-page alternatives.
## Core Framework
Interaction design is what the product feels like to use — not how it looks, but how it responds, how it communicates, and how it makes the user feel in every moment.
## Principle
Every design decision should be traceable to a user need or a business objective. If the rationale for a decision is "it's best practice" or "it looks good," the decision hasn't been justified — it's been assumed.
## When to use
When this specific UX challenge is the active constraint.
